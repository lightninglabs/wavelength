# sdk/swaps/sqlc

## Purpose

Generated type-safe database access for `sdk/swaps`'s isolated SQLite
store: exposes `pay_swaps` and `receive_swaps` as typed Go structs plus
one query method per operation the pay/receive FSMs need (get, list,
list-pending, upsert). Contains no business logic — it is pure
generated plumbing between `sdk/swaps.Store` and the underlying `DBTX`.

## Key Types

- `Queries` — holds a `DBTX` handle; the concrete implementation of
  `Querier`. Constructed via `New(db DBTX)`; `WithTx(tx *sql.Tx)`
  returns a copy bound to a transaction.
- `DBTX` — minimal interface (`ExecContext`, `PrepareContext`,
  `QueryContext`, `QueryRowContext`) satisfied by both `*sql.DB` and
  `*sql.Tx`, letting callers run queries inside or outside a
  transaction.
- `Querier` — interface enumerating every generated query method:
  `GetPaySwap`, `GetReceiveSwap`, `ListPaySwaps`, `ListPendingPaySwaps`,
  `ListPendingReceiveSwaps`, `ListReceiveSwaps`, `UpsertPaySwap`,
  `UpsertReceiveSwap`. `var _ Querier = (*Queries)(nil)` pins `Queries`
  to the interface at compile time.
- `PaySwap` — row/model for one Ark-to-Lightning pay session: payment
  hash, invoice, fee/amount fields, FSM `State`, negotiated delay and
  pubkey fields, vHTLC identity (`VhtlcPkscript`, `VhtlcPolicyTemplate`,
  `VhtlcOutpoint`, `VhtlcAmount`), funding/refund session and recovery
  IDs, `Preimage`, `InterventionReason`, `SettlementType`, timestamps.
- `ReceiveSwap` — row/model for one Lightning-to-Ark receive session;
  same vHTLC-identity and delay/pubkey shape as `PaySwap`, plus
  receive-specific fields (`PaymentAddr`, `PendingHtlcAckCursor`,
  `ClaimReceivePubkey`/`ClaimReceivePkscript`, `ClaimSessionID`,
  `ClaimRecoveryID`) and credit-assist planning fields
  (`RequestedAmountSat`, `AvailableCreditSat`, `AttachedCreditSat`,
  `DustLimitSat`).
- `UpsertPaySwapParams` / `UpsertReceiveSwapParams` — parameter structs
  for the corresponding `:exec` upsert queries (insert with
  `ON CONFLICT (payment_hash) DO UPDATE`).

## Relationships

- **Depends on**: `database/sql` only (no other darepo-client package
  imports). Query and schema source lives in `sdk/swaps/queries/swaps.sql`
  and `sdk/swaps/migrations/*.sql`.
- **Depended on by**: `sdk/swaps` (`store.go`, `store_sessions.go` wrap
  `*sqlc.Queries` to load/persist `PaySession` and `ReceiveSession`
  state, including the vHTLC identity fields consumed by the
  forfeit-signature validation path and the receive-side credit-assist
  plan).

## Invariants

- This package is generated by `sqlc` from `sdk/swaps/migrations/`
  (schema) and `sdk/swaps/queries/swaps.sql` (queries), per the
  `sdk/swaps` entry in the top-level `sqlc.yaml`. Never hand-edit;
  regenerate via `make sqlc` and commit the generated diff separately.
- No business logic belongs here — validation, FSM transitions, and
  amount/script matching all live in `sdk/swaps`, not in this package.
- All DB access goes through the `DBTX` abstraction so callers can run
  queries against either a bare connection or an active transaction
  without duplicating query bodies.
- Column additions (e.g. the vHTLC identity columns or the receive
  credit-assist columns) arrive as new migration files plus a `make
  sqlc` regen; `PaySwap`/`ReceiveSwap` field sets and `Querier` method
  signatures must always match the current migrated schema.
