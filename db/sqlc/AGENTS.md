# db/sqlc

## Purpose

Generated sqlc query layer providing type-safe Go wrappers over all SQL
queries used by the darepo server. Most files are auto-generated from
`db/queries/`; do not edit them. The custom file `db_custom.go` adds
non-generated helpers for multi-backend support (SQLite vs Postgres).

## Key Types

- `Querier` — Generated interface listing every database query function;
  implemented by `*Queries`.
- `Queries` — Concrete query executor bound to a `DBTX` connection.
- `BackendType` — Enum (`BackendTypeSqlite`, `BackendTypePostgres`) stored
  inside `wrappedTX` so queries can emit backend-specific SQL at runtime.
- `wrappedTX` — Internal wrapper around `DBTX` that carries the backend type;
  used by `Queries.Backend()` to detect which dialect is active.
- `NewWithBackend` / `NewSqlite` / `NewPostgres` — Constructors that wrap a
  connection with the appropriate `BackendType`.
- `models.go` — Generated struct types mapping 1-to-1 with database tables
  (`Vtxo`, `Round`, `OorSession`, `MailboxEnvelope`, etc.).

## Relationships

- **Depends on**: nothing outside the standard library and database/sql.
- **Depended on by**: `db` (wraps `*Queries` in store adapters), `db/sqlc/migrations`
  (schema migrations co-located here).

## Invariants

- All `*.sql.go` files are generated by sqlc; never edit them directly.
- To add or modify queries, edit `db/queries/*.sql` and run `make sqlc`.
- `BackendType` must be set via `NewSqlite` / `NewPostgres` / `NewWithBackend`;
  passing a bare `DBTX` to `New()` leaves the backend unknown and disables
  the dialect check in `Backend()`.
- `makeQueryParams` is a workaround for the sqlc `/*SLICE:<field>*/` pattern;
  it produces numbered Postgres-style placeholders (`$1`, `$2`, …) from a
  flat parameter count.
