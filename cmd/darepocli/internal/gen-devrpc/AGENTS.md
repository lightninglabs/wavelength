# cmd/darepocli/internal/gen-devrpc

## Purpose

Build-time code generator that produces
`cmd/darepocli/darepoclicommands/devrpc/registry_generated.go`. Scans
`daemonrpc`, `swapclientrpc`, `walletdkrpc`, and `btcwallet/rpc/walletrpc`
proto descriptors and emits a static service/method registry with alias
tables and extracted comments for the dev RPC command tree.

## Key Types

- `main()` — Entry point: collect services, render the registry function,
  apply `go/format`, write the output file.
- `serviceData` — Build-time service representation: `FullName`, `Aliases`,
  `Methods`, `Comments`.
- `methodData` — Build-time method representation: `Name`, `Aliases`,
  `Input`, `Output`, `ClientStreaming`, `ServerStreaming`, `Comments`.

## Design

- **Build-time only.** Run via Makefile (or `go generate`) to refresh the
  generated registry. The output is committed so the binary does not depend
  on proto reflection at runtime.
- **Alias generation.** Method aliases are auto-generated by converting
  CamelCase to kebab-case with initialism normalization (`VTXO` → `Vtxo`,
  `UTXO` → `Utxo`, `OOR` → `Oor`, `ECDH` → `Ecdh`, `TXID` → `Txid`). Service
  aliases are hardcoded in `serviceAliases()`: `daemon`, `swapclient`,
  `wallet` (`walletdkrpc.WalletService`), `wallet-inspection`
  (`walletdkrpc.WalletInspectionService`), `btcwallet-version`, and
  `btcwallet`.
- **Comment extraction.** Proto `SourceLocation.LeadingComments` are
  preferred; falls back to a hand-written proto file parser when location
  data is absent.

## Relationships

- **Depends on**: `daemonrpc` (for `File_daemon_proto`),
  `rpc/swapclientrpc` (for `File_swap_client_proto`), `rpc/walletdkrpc`
  (for `File_wallet_proto`), `btcwallet/rpc/walletrpc` (for
  `File_api_proto`).
- **Depended on by**: nothing at runtime. This is a `package main` build
  tool invoked by `make rpc` or an equivalent Makefile target.

## Invariants

- The output file contains a "Code generated by gen-devrpc. DO NOT EDIT."
  header; editing it manually will be overwritten on the next `make rpc`.
- The generator validates every entry in `expectedServices` — currently
  `daemonrpc.DaemonService`, `swapclientrpc.SwapClientService`,
  `walletdkrpc.WalletService`, `walletdkrpc.WalletInspectionService`,
  `walletrpc.VersionService`, and `walletrpc.WalletService` — against the
  scanned files; it fails with an error if any is missing, catching proto
  renames early. Comment extraction (`protoCommentFiles`) only covers
  `daemonrpc` and `swapclientrpc`, so `walletdkrpc`/`walletrpc` methods rely
  solely on `SourceLocation.LeadingComments`.
