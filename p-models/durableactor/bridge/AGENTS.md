# p-models/durableactor/bridge

## Purpose

Test-only Go bridge between the P formal model for the durable-actor mailbox
and the production `db/actordelivery` SQLite implementation. Parses P model
trace files (JSON), replays each operation sequence against a real SQLite
store, and asserts the expected outcome — ensuring the production
implementation conforms to the formally verified specification.

## Key Types

- `MailboxTrace` — Parsed representation of a single P model trace file.
  Fields: `Events []MailboxTraceEvent`. Constructed via `ParseMailboxTrace(path)`
  or `ParseMailboxTraceDir(dir)` for a whole directory.
- `MailboxTraceEvent` — One step in a trace: operation name (enqueue, lease,
  ack, nack, commit, dead_letter, expire_leases) plus operation-specific
  parameters and the expected row counts after the operation.
- `ReplayMailboxTrace(t *testing.T, trace *MailboxTrace)` — Replays a parsed
  trace against a fresh in-memory SQLite database, asserting expected row
  counts after each step. Called by the package's own
  `TestReplayMailboxModelTraces` which drives every `.json` file under
  `../traces/`.

## Relationships

- **Depends on**: `db/actordelivery` (production SQLite delivery store
  under test), `testing` (test-only package).
- **Depended on by**: nothing (no production code imports this package; it
  is only invoked by its own test suite).

## Invariants

- The package is test-only: no production binary should import it.
- Each `ReplayMailboxTrace` call creates an isolated in-memory SQLite
  database so traces are independent and parallelism-safe.
- Trace files live in `../traces/` (relative to this package) and are
  generated by the P model's trace-export mechanism, not written by hand.
